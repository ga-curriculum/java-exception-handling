## About

Exceptions are runtime errors that can occur during program execution. Such exceptions can crash applications, and error messages that the Java Runtime Engine (JRE) throws on the screen can leave its non-technical end users in a lurch. A good programmer will pre-empt such exceptions and handle them in their code itself. When an exception occurs, exception handling code does either one or both of the following:
1. Prevents an application crash.
2. Provides an end-user-friendly communication about the exception: 
   - Taking corrective action (For example, informing the user that the user ID or password they entered is wrong and asking them re-enter the correct credentials)
   - Informing the user with some meaningful information (For example, informing that a service is down for scheduled maintenance and they can try using it after a specified time)

In this lesson you will learn about all the exception handling options available in Java
